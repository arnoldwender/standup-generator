# Contributing to Standup Generator

Welcome! Yesterday I reviewed this contributing guide. Today I'm finishing it up. No blockers.

(See? You're already getting the hang of it.)

## Getting Started

```bash
git clone https://github.com/arnoldwender/standup-generator.git
cd standup-generator
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173). If it works, you're already more productive than most standups.

## Submitting a Pull Request

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Make sure the app still builds (`npm run build`)
4. Open a PR with a clear description — pretend it's a standup update, but one that actually says something useful
5. Wait for review — we promise it'll take less time than your average sprint retrospective

## Code Style

- **TypeScript** — strongly typed, unlike most standup updates
- **React** functional components with hooks
- Keep the corporate satire sharp but relatable — we've all been in that meeting
- New buzzwords must be genuinely funny, not just random words. "Synergize the blockchain" is tired. Be better.

## Bug Reports

Open an issue with:
- What you expected to happen
- What actually happened
- Your browser and OS
- Your current buzzword density preference (for science)

## Feature Requests

Got an idea for a new developer role, achievement, or standup phrase? Open an issue! The best contributions come from real meeting trauma.

## The Golden Rule

If your PR could be summarized as "moving tickets across the board," it's probably not ready yet. Make it count.
